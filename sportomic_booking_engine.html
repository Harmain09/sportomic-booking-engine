How the Engine Works

PHYSICAL LAYOUT:
|<─── T1_7v7 (Left Half) ───>|<─── T2_7v7 (Right Half) ───>|
|<─ T1_5v5 ─>|<─ T2_5v5 ─>|<─ T3_5v5 ─>|
OVERLAP MAP - main data structure:
T1_7v7 → blocks [ T1_5v5, T2_5v5 ]
T2_7v7 → blocks [ T2_5v5, T3_5v5 ]
T1_5v5 → blocks [ T1_7v7 ]
T2_5v5 → blocks [ T1_7v7, T2_7v7 ]  ← middle blocks BOTH 7v7!
T3_5v5 → blocks [ T2_7v7 ]
ALGORITHM:
1. For each booked court → add its overlapping courts to BLOCKED set
2. booked = "Booked"  |  in blocked = "Blocked"  |  else = "Available"
